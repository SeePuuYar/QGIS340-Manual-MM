24.1. GNSS/GPS data များကိုမိတ်ဆက်ခြင်း (Introducing GNSS/GPS Data)

  • GPS ဆိုတာဘာလဲ

  • GPS data များ ရွေ့ပြောင်းခြင်း/ခေါ်ယူထည့်သွင်းခြင်း (Transferring or loading GPS data)

    • GPX file တစ်ခုထည့်သွင်းခြင်း (Loading a GPX file)

    • Device တစ်ခုထဲသို့ ထည့်သွင်းခြင်း သို့မဟုတ် Device တစ်ခုမှ ထည့်သွင်းခြင်း (Loading to or from a device)

24.1.1. GPS ဆိုတာဘာလဲ

GPS Global Positioning System ဆိုသည်မှာ GPS receiver ရှိသော မည်သူမဆို ကမ္ဘာကြီးရဲ့ မည်သည့်နေရာတွင်ရှိနေသည်ဖြစ်စေ ဂြိုလ်တုများကို အသုံးပြုပြီး ၎င်းတို့၏တိကျသော တည်နေရာကို ဖော်ပြပေးနိုင်သော စနစ်ဖြစ်ပါသည်။ GPS ကို လမ်းညွှန်ပြသရာတွင် ထောက်ကူပစ္စည်းအဖြစ် အသုံးပြုပါသည်။ ဥပမာ - လေယာဉ်၊ လှေ နှင့် တောင်တက်သူများ။ GPS receiver သည် ၎င်းရဲ့ လတ္တီကျု၊ လောင်ဂျီကျု နှင့် (တခါတရံ) အမြင့်တို့ကို တွက်ချက်ရန်အတွက် ဂြိုလ်တုများမှ အချက်ပြမှုကို ဖမ်းယူလုပ်ဆောင်ပါသည်။ Receiver အများစုသည် အောက်ပါအချက်အလက်များကို သိမ်းဆည်းထားနိုင်စွမ်းရှိပါသည်-

  • တည်နေရာများ (waypoint များဟုခေါ်ပါသည်)

  • စီစဉ်ထားသော route (လမ်းကြောင်း) တစ်ခုကိုဖြစ်ပေါ်စေသော တည်နေရာအဆက်များ

  • အချိန်နှင့်အမျှ receiver ၏ ရွှေ့လျားခဲ့သော track (ခြေရာခံလမ်းကြောင်း) မှတ်တမ်း

Waypoint များ၊ route များ နှင့် track များတို့သည် GPS data ထဲတွင် အခြေခံ feature အမျိုးအစား ၃ မျိုးဖြစ်ပါသည်။ QGIS သည် waypoint များကို point layer များအဖြစ် ဖော်ပြပြီး routes နှင့် tracks များကို linestring layer များအဖြစ် ဖော်ပြပါသည်။

Note

QGIS သည် GNSS receiver များနှင့်လည်း အလုပ်လုပ်နိုင်ပါသည်။ သို့သော် ဤ documentation တွင် GNSS အစား GPS ဆိုသည့် အသုံးအနှုန်းကို ဆက်လက်သုံးစွဲသွားပါမည်။

24.1.2. GPS data များ ရွေ့ပြောင်းခြင်း/ခေါ်ယူထည့်သွင်းခြင်း (Transferring or loading GPS data)

24.1.2.1. GPX file တစ်ခုထည့်သွင်းခြင်း (Loading a GPX file)

GPS data များကိုသိမ်းဆည်းခြင်းအတွက် file format အမျိုးအစားများစွာရှိပါသည်။ QGIS မှ အသုံးပြုသော format မှာ GPX (GPS eXchange format) ဖြစ်ပြီး file တစ်ခုတည်းတွင် waypoint များ၊ route များ နှင့် track များပါဝင်သော ဖလှယ်နိုင်သည့် standard (စံ) format တစ်ခုဖြစ်ပါသည်။

GPX file တစ်ခုကို ခေါ်ယူထည့်သွင်းရန်အတွက်-

  1. Data Source Manager dialog ထဲမှ GPS tab ကို ဖွင့်ပါ။ ဆိုလိုသည်မှာ- * Toolbar ပေါ်ရှိ Open Data Source Manager ခလုတ်ကိုနှိပ်ပြီး (သို့မဟုတ် Ctrl+L ကိုနှိပ်ပါ) target tab ကို အသုံးပြုနိုင်ရန်ဖွင့်ပါ။ * သို့မဟုတ် Layer ► Add Layer ► Add GPX Layer… ကိုရွေးချယ်ပါ။

  2. GPX file ကိုရွေးချယ်ရန် GPX dataset ရဲ့ဘေးရှိ … Browse ခလုတ်ကိုအသုံးပြုပါ

  3. File မှတဆင့် ခေါ်ယူထည့်သွင်းလိုသော Feature types (Feature အမျိုးအစား) ကိုရွေးချယ်ရန် check box (အမှန်ခြစ်ခြစ်ရသောနေရာများ) ကိုအသုံးပြုပါ။ Feature အမျိုးအစား တစ်မျိုးချင်းစီ (Waypoints၊ Tracks သို့မဟုတ် Routes) များကို သီးခြား layer တစ်ခုချင်းစီတွင် ခေါ်ယူထည့်သွင်းပါလိမ့်မည်။

24.1.2.2. Device တစ်ခုထဲသို့ ထည့်သွင်းခြင်း သို့မဟုတ် Device တစ်ခုမှ ထည့်သွင်းခြင်း (Loading to or from a device)

မတူညီသော GPS ကိရိယာအမျိုးအစားများနှင့် format များ အများအပြားရှိကြပါသည်။ QGIS သည် GPX ဖိုင်များကိုအသုံးပြုသောကြောင့် အခြား GPS ဖိုင် format များမှ GPX သို့ပြောင်းလဲပေးရန် နည်းလမ်းလိုအပ်ပါသည်။ QGIS တွင် GPSBabel ဆိုသည့် အခမဲ့ program ကိုအသုံးပြုပြီး ထိုအရာကိုလုပ်ဆောင်နိုင်ပါသည်။ GPSBabel သည် Garmin သို့မဟုတ် magellan ကဲ့သို့ လူသုံးများသော GPS ကိရိယာများနှင့် Google Earth သို့မဟုတ် Basecamp ကဲ့သို့ mapping program များအကြား waypoint များ၊ track များနှင့် route များကို ပြောင်းလဲပေးနိုင်ပါသည်။ ရာချီသော GPS ကိရိယာများနှင့် program များကို ထောက်ပံ့ပေးထားပါသည်။ ၎င်းသည် သင့်ကွန်ပျူတာနှင့် GPS ကိရိယာအကြား GPS data များကိုလည်း ကူးပြောင်းပေးနိုင်ပါသည်။

Settings ► Options ► GPS ► GPSBabel အောက်တွင် သင့် GPS ကိရိယာအမျိုးအစားကိုသတ်မှတ်ပေးနိုင်ပြီး နောက်ပိုင်းတွင် Processing GPS algorithms မှအသုံးပြုမည့် conversion parameter များကိုသတ်မှတ်ပေးနိုင်ပါသည်။

  1. ပထမဦးစွာ Path to GPSBabel binaries များကို သတ်မှတ်ပေးရပါမည်။

  2. ထို့နောက် သင့် ကိရိယာကို ထည့်သွင်းပါ။ Add new device သို့မဟုတ် Remove device ခလုတ်ကိုအသုံးပြုပြီး ကိရိယာစာရင်းကို update လုပ်နိုင်ပါသည်။

  3. ကိရိယာတစ်ခုချင်းစီအတွက်-

    • Device name တစ်ခုပေးပါ

    • ကိရိယာနှင့်အပြန်အလှန်လုပ်ဆောင်ရာတွင် QGIS မှအသုံးပြုမည့် မတူညီသော Commands များကိုသတ်မှတ်ပေးပါ၊ ဥပမာ-

      • Waypoint download - ကိရိယာမှ waypoint များဒေါင်းလုဒ် ပြုလုပ်ခြင်း

      • Waypoint upload - ကိရိယာထဲသို့ waypoint များ ထည့်သွင်းခြင်း

      • Route download - ကိရိယာမှ route များ ဒေါင်းလုဒ် ပြုလုပ်ခြင်း

      • Route upload - ကိရိယာထဲသို့ route များ ထည့်သွင်းခြင်း

      • Track download - ကိရိယာမှ track များ ဒေါင်းလုဒ် ပြုလုပ်ခြင်း

      • Track upload - ကိရိယာထဲသို့ track များ ထည့်သွင်းခြင်း

      များသောအားဖြင့် command များသည် GPSBabel command များဖြစ်သော်လည်း GPX ဖိုင်ဖန်တီးပေးနိုင်သော အခြား command line program တစ်ခုခုကိုလည်း အသုံးပြုနိုင်ပါသည်။ QGIS သည် command ကို run သောအခါ %type ၊ %in ၊ နှင့် %out keyword များကို အစားထိုးပေးပါလိမ့်မည်။

      ဥပမာတစ်ခုအနေဖြင့်- ကိရိယာအမျိုးအစားတစ်ခုကို download command gpsbabel %type -i garmin -o gpx %in %out ဖြင့်ဖန်တီးပြီးနောက် ၎င်းကို port /dev/ttyS0 မှ output.gpx ဖိုင်သို့ waypoint များ download ပြုလုပ်ရန် အသုံးပြုပါက QGIS သည် keyword များကို အစားထိုးပေးပြီး gpsbabel -w -i garmin -o gpx /dev/ttyS0 output.gpx command ကို run ပေးပါလိမ့်မည်။

      သင်အသုံးပြုလိုသည့် သီးသန့် command line option များကို GPSBabel manual တွင် ဖတ်ရှုပါ။

ကိရိယာအမျိုးအစားအသစ်တစ်ခုကို ဖန်တီးပြီးသည်နှင့် ၎င်းသည် GPS download နှင့် upload algorithm များလုပ်ဆောင်ရန်အတွက် ကိရိယာများစာရင်းထဲတွင် ပေါ်နေပါလိမ့်မည်။

Note

GPS များတွင် data များကို coordinate system အမျိုးမျိုးဖြင့် သိမ်းဆည်းထားနိင်ပါသည်။ GPX file တစ်ခုကို (GPS သို့မဟုတ် web site မှ) download ရယူပြီး QGIS ထဲသို့ ခေါ်ယူထည့်သွင်းသောအခါ GPX file ထဲတွင် သိမ်းဆည်းထားသော data များသည် WGS 84 (latitude/longitude) ဖြစ်နေရန်လိုအပ်ပါသည်။ QGIS သည် ထိုကဲ့သို့ဖြစ်မည်ဟု မျှော်လင့်ထားပြီး ၎င်းသည် တရားဝင် GPX specification (သတ်မှတ်ချက်) ဖြစ်ပါသည်။ GPX 1.1 Schema Documentation <https://www.topografix.com/GPX/1/1/> တွင်ကြည့်ရှုပါ။
